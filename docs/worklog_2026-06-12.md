## 五、实测排障（最大的工作量，全部有源码佐证）

### MUD 侧（基础设施修复，游戏数值零改动）
1. **libevent-2.0.so.5 缺失** → 用户目录符号链接垫片（`tools/start_mud.sh`）。
2. **双 driver 并发写 swap → 运行时腐化**（eval_cost 爆炸、拒绝连接）
   → start_mud.sh 单实例守卫 + 清理 swap 残留。
3. **`maximum evaluation cost` 30M→300M**：守护进程被清理后重载编译链
   超 30M 指令 → 第一连接必被拒。属基础设施参数。
4. **`is_chinese()` 按码点语义编写、驱动是字节语义 → 全服无法创建角色**
   （任何中文名被拒）→ 改为 UTF-8 字节序列检测。这是存量 bug，不修无人能建号。
5. 中文名长度按字节校验（≤6 字节=2 汉字）→ 凭据生成器适配。

### 机制真相（逐行核验 combatd.c / learn.c / npc.c，推翻多项设计假设）
- **切磋(fight)一击散场**：任一方造成有效伤害即"承让"，每次接战仅 2-4 秒
  → 纯切磋实测 5~60 exp/h，不可用作主引擎。
- **kill 真打不散场**：每心跳互攻，经验三通道持续判定 → 主引擎改为 kill 模式。
- **经验通道全是"以弱对强"**：命中且 ap<dp（77%）/ 闪避且 dp<ap（61%）/
  被重击 random(max_kee+kee)<damage。打弱者无收益。
- **出手频率公式** `random(victim_cps×3) < attacker_cor + bellicosity/50`
  → 野兽 cps 5~15（人类 10~30）→ 打野兽出手频率约 2 倍（印证用户经验）。
- **NPC 隐藏属性普遍**：set_temp("apply/dodge")（妇人实际 dp≈600 而非估算 25）、
  战斗中 `command("wield knife")` 动态武装、chat_msg_combat 吃攻击轮
  → **静态估算不可靠，改为运行时实测 bandit 选目标**（EWMA 速率库 +
  速率底线轮换 + 拉黑带 TTL）。
- **kill 不走 accept_fight**：friendly 拒战只影响切磋 → 阶梯全面放开
  （59 → 128 候选，含群兽点）。
- **NPC 反向成长**：NPC 打不中强者时自己 +exp +技能（`!userp` 专属通道）
  → 陪练自动变强，长期可持续。
- **任务系统半成品（印证用户怀疑）**：寻物类 `accept_object` 整段被注释；
  杀类结算（killer_reward）是活代码、15 张表 253 个任务全为杀类 → 杀类理论可用，
  exp>1100 后实测验证。
- 金疮药 2000 文（前期买不起）→ 买药失败 30 分钟冷却，伤势靠自然恢复
  （气满后 eff_kee +1/tick）。

### 智能体侧典型 bug（实测暴露）
- 登录：同提示循环守卫（曾把名字当命令刷爆 logind）；断线重连服务器
  **直接附身进游戏**（无登录提示）→ hp 探测识别。
- `logged_in` 是连接级状态却被持久化 → 重连后没登录就跑例程被踢
  → 每次重连强制重新登录。
- **测试污染生产数据**：planner 测试把 exp=3000 假状态写进真 checkpoint
  → `tests/conftest.py` 重定向到 /tmp。
- checkpoint 旧时间戳 → 重启即判"硬停滞"fatal → 恢复时补续点。
- SIGTERM 优雅退出码 0 被 watchdog 当"完成"→ 退出码语义重定义
  （0=达标，2=可重启）。
- 多攻击者围殴致死（6×trainee 秒杀 100 气新手）→ 攻击者数按 max_kee 动态限制
  （每 120 气 +1）、撤退线 60%、wimpy 40。
- 工具链自杀陷阱 ×3：pkill/pgrep -f 模式匹配到自身命令行 → `agent[.]py`
  写法 + 杀进程与启动分离执行。
- `entry_count_for` 补丁丢失 → **意外实战验证了升级通道**：例程异常 →
  planner 派发修复任务 → DeepSeek 接管探路 → 全自动，架构按设计工作。

## 九、深度挖掘：经验系统全貌（21:30 второй调研）

应用户要求穷举式扫描 mudlib 全库经验机制。Explore 子代理 + 本人复核，
产出按"直接来源/间接加速"分类的完整清单。核心新发现与落实：

### 已实施
1. **gin/sen 效率线**（spar）：经��判定 `random(gin%+int)>30/50` → gin 满 77%
   通过率、半血 61%；`skill_power ∝ sen 当前值` → sen 低全面降效。
   现 gin/sen <65% 即撤退恢复，恢复目标 85%。
2. **识字回路**（maintain）：literate 每 10 级 +2 int（唯一可刷的 int 来源），
   int 永久提升经验判定/learn 增量/study 效率。书生处 kill 取书（书生 exp=10
   wimpy=100 无威胁）→ study 到书上限 50 级（约 +10 int）。自举：literate
   越高 study 增量越大（literate/5+1）。
3. **exercise 内力闭环**（maintain）：气→内力（30气≈+force 数点），
   内力是 exert recover 的燃料 → 战斗续航闭环：exercise 攒内力 → 战斗 →
   recover 秒回气 → 减少恢复停机。
4. **洗杀气**（maintain）：kill 流杀气 +1/杀；杀气/40 > cps 时 NPC 会 berserk
   主动攻击。>80 时去雪山寺 give 200 coin to keeper 洗（按 kar 随机减）。
5. **任务潜能管理**（quest）：potential 入账硬顶 learned+100 → 接单前余量
   >60 先去师父处 learn 倾泻，保 pot_bonus 全额入账。连胜 tfinished 计数
   （每 3 连胜升档），超时重领会清零——已记录 streak 计数器。
6. **审判官保险加固**：答错 4 次即死。本地正则解四则/最大公因数（中文数字），
   LLM 仅兜底，确保 20 秒限时内应答。

### 重要结论（核验否决项）
- 补品系统是死代码（eat_func 注释，人参/灵芝 tonic 不生效）→ 无食物加经验捷径
- quest_factor 恒 ×1.0（1.5 倍通道被注释）→ 不用追
- 连击无 attack_count；biwu 多回合仅限玩家对玩家 → 不可用
- busy 流（counterattack 把对手打 busy → dp/3 + 每心跳必攻）需 swordsman
  独门技能，本角色已拜柳淳风（swordsman 系）→ **中期可解锁**，记入待办
- 战斗中 get/eat/steal/自 busy perform 会反被 quick-attack 连打 → 例程已确认无此类动作
